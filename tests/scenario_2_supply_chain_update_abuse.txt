A regional engineering firm with offices in Maryland and Washington, D.C. noticed unusual outbound connections from several project management servers after installing a routine software update from a trusted vendor. The update package had been downloaded from the vendor’s official portal and was digitally signed, so the installation initially raised no alarms.

Three days later, network defenders discovered that the updated application was launching a hidden child process that contacted an external command-and-control server over HTTPS using traffic that blended in with normal software telemetry. The malicious component collected hostnames, usernames, operating system versions, and installed security tools from infected systems. In at least two cases, the malware executed PowerShell commands to enumerate domain trusts and query Active Directory group memberships.

Further analysis revealed that the attackers had compromised the vendor’s build environment and inserted a backdoor into a legitimate software update. The malware only activated on systems belonging to organizations in the defense, energy, and engineering sectors. It used a kill switch to remain dormant in non-target environments.

On the affected firm’s network, investigators found evidence that the backdoor staged compressed archives containing engineering drawings and contract documentation in a temporary directory before transmitting them in small encrypted chunks to attacker-controlled infrastructure. DNS logs also showed brief communication with fallback domains likely intended for resilience if the main command server was taken down.

Incident responders concluded that the actor was likely conducting espionage rather than destructive operations. Attribution remains uncertain, but the tradecraft suggests a well-resourced advanced threat actor with knowledge of software supply chain compromise techniques.
