A public utility contractor supporting water treatment facilities in the Midwest reported a security incident after an engineer used a USB drive received during an industry conference. The device was labeled as containing product brochures and maintenance manuals from a vendor booth. When connected to a Windows laptop used for field diagnostics, the USB drive automatically presented a PDF document and a folder of legitimate-looking files.

Unknown to the engineer, the USB also contained a malicious shortcut file and an executable disguised with a double extension. Within minutes, the laptop began spawning command-line processes that created a scheduled task for persistence and copied files into an obscure directory under ProgramData. Endpoint alerts later showed that the malware attempted to disable local antivirus protections and queried the system for connected network shares.

Over the next day, the infected laptop connected back to an attacker-controlled server through TCP port 443, but the traffic did not match normal browser behavior. The malware received commands to list files, collect configuration documents, and capture screenshots. Analysts also found evidence that the infected host attempted to authenticate to a file server used to store facility diagrams, maintenance schedules, and remote access instructions for operational technology support teams.

No direct disruption to water operations occurred, but investigators are concerned because the compromised laptop had periodic access to sensitive industrial control system documentation. The initial infection vector appears to be removable media, and the attacker demonstrated an interest in reconnaissance and collection rather than immediate sabotage. The identity of the actor is unknown.
